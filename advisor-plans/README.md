## Findings considered and rejected (so they aren't re-audited)

- **Event-store "sequence collision" race** (reported 4 ways): not a bug. Event `sequence` is scoped per `runId` (`countRunEvents` filters `WHERE runId = ?`, reads order `WHERE runId=? ORDER BY sequence`). Distinct runs — including parent vs. child — legitimately reuse sequence values; they never collide.
- **"Duplicate assistant bubbles" on approval-resume** (the prior audit's F3): stale. Its merge function and `agent-chat-projection.ts` were deleted in the pivot; the current `build-chat-stream-response.ts:509` `onFinish` persists the Mastra bridge's `nextMessages` directly with no `originalMessageCount` boundary.
- **Delegation `.catch` swallows errors** (`delegation.ts:294`): by design — the secondary record-failure is logged while the original error is rethrown at `:299`.
- **Delegation semaphore goes negative on failed acquire**: false — the failed-acquire `throw` (`:247`) is _outside_ the `try/finally`, so the slot is released only after a successful acquire (and `releaseChildSlot` has a `Math.max(0, …)` guard anyway).
- **`model-message-continuity.ts:262` assigns to index −1**: false — the `if (previousMessage)` guard is present (`:264`).
- **Approval re-resolution across runs in a session** (`recordAgentRunOutcome`): by design — resolving a pending approval by `toolCallId` across the session is the approve-after-restart mechanism; collision needs two runs sharing a random `toolCallId`, which is effectively impossible.
- **Symlink TOCTOU / workspace escape on read & write** (`workspace-core.ts`): containment is sound against the realistic threat — `path.resolve` + `path.relative` rejects `..`/absolute, the ancestor is `realpath`-checked against the project root, and `lstat`/`!isFile()` rejects symlink _targets_. The residual TOCTOU needs a local attacker racing the filesystem; tracked only as a low-priority hardening note + the missing write-through-symlink test.
- **Local-server CSRF / missing auth** (SEC-06): mitigated — every `/rpc/*` and `/api/*` route requires a per-launch 256-bit bearer token; CORS is localhost-only.
- **Event-read N+1 / projection rebuild cost** (PERF): rejected — the schema has `agent_events_run_sequence_idx` on `(run_id, sequence)` and a run's events are bounded to one turn. (`countRunEvents` using `select(...).length` instead of `COUNT(*)` is a trivial cleanliness nit, not worth a plan.)
- **`serialize` / `safeParse` silently return null** (`agent-event-store.ts`): by design — defensive tolerance for an evolving-payload event store.
- **Post-pivot orphan/dead code**: the pivot was clean — the only reference to any deleted module across `src`+`test` is one string literal in a test fixture.
- **Missing `.env.example`**: provider keys live in `electron-store` (in-app Settings), not `.env`; not a real onboarding gap for this desktop app.
- **Electron `webPreferences` not hardened explicitly** (SEC-04): safe via Electron 41 defaults (`contextIsolation:true`, `nodeIntegration:false`, `sandbox:true`). Low-priority hardening only.
